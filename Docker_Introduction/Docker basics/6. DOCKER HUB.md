# Sử dụng Docker Hub

Sau khi đã có image, Docker Hub là nền tảng để chia sẻ các image này, tương tự như GitHub hay GitLab cho mã nguồn.

## 1. Tạo Repository trên Docker Hub

Đầu tiên, bạn cần tạo một repository trên Docker Hub. Tên của repository sẽ có cấu trúc: `YOUR-USER-NAME/repo-name`.

*Ví dụ:* `phuctran362003/library-manager`

## 2. Gắn Tag cho Image (docker tag)

Sử dụng lệnh `docker tag` để đánh dấu (tag) image cục bộ của bạn với tên repository trên Docker Hub. Điều này cho Docker biết rằng bạn sẽ push image này lên repository đó.

```bash
docker tag library-manager YOUR-USER-NAME/library-manager
```

## 3. Push Image lên Docker Hub (docker push)

Sau khi đã gắn tag, bạn có thể push image lên Docker Hub bằng lệnh `docker push`.

```bash
docker push phuctran362003/library-manager
```

## 4. Đăng nhập Docker Hub (Nếu cần)

*Lưu ý:* Nếu bạn gặp lỗi `unauthorized`, bạn sẽ cần phải đăng nhập vào Docker Hub. Thao tác này chỉ cần thực hiện một lần duy nhất trên máy của bạn.

```bash
docker login -u <DOCKERHUB-USERNAME>
```

*Ví dụ:* `docker login -u phuctran362003`

Sau đó, nhập mật khẩu của bạn khi được yêu cầu và thử lại bước 3.

## 5. Xác nhận Thành công

Nếu quá trình push thành công, bạn sẽ thấy output tương tự như sau:

```
The push refers to repository [docker.io/phuctran362003/library-manager]
...
latest: digest: sha256:... size: ...
```

## 6. Deploy Phiên bản Free (Play With Docker)

Sau khi image đã được push lên Docker Hub, bạn có thể chia sẻ ứng dụng của mình để người khác có thể truy cập. Dưới đây là cách triển khai trên nền tảng miễn phí *Play With Docker*.

*Lưu ý:* Đây là phiên bản miễn phí, mỗi session chỉ kéo dài 4 tiếng. Nếu bạn muốn thời gian hoạt động lâu hơn, hãy cân nhắc thuê máy chủ VPS.

**Bước 1:** Truy cập vào [Play With Docker](https://labs.play-with-docker.com/)

**Bước 2:** Đăng nhập bằng tài khoản Docker Hub của bạn.

**Bước 3:** Nhấn "Add Instance" để tạo một phiên bản mới.

**Bước 4:** Gõ lệnh sau vào terminal trên Play With Docker:

```bash
docker run -dp 3000:8080 <YOUR-USER-NAME/getting-started>
```

*   `<YOUR-USER-NAME/getting-started>`: Là tên của image sau khi bạn đã push lên Docker Hub.
*   `3000:8080`: Cổng bên phải (`8080`) *bắt buộc* phải khớp với cổng mà ứng dụng trong container đang lắng nghe. Cổng bên trái (`3000`) là cổng trên máy chủ Play With Docker mà bạn sẽ truy cập.

**Bước 5:** Chờ một lát để hệ thống tải (pull) image từ Docker Hub về máy chủ cloud.

**Bước 6:** Sau khi container khởi động, bạn sẽ thấy output tương tự như sau:

```
... (output của lệnh docker run) ...
```

**Bước 7:** Nhấn "Open Port" (thường xuất hiện ở phía trên terminal) và nhập cổng bạn đã ánh xạ (ví dụ: `3000`) để truy cập ứng dụng web của bạn.
