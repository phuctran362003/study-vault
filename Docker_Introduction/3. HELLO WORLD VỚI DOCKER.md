# Các khái niệm cơ bản và thuật ngữ của Docker

## 1. Container (hộp cơm)

**Là cái gì?** Container là một ứng dụng hoặc dịch vụ được đóng gói trong một môi trường nhẹ. Nó chạy độc lập và chứa tất cả những thứ cần thiết để nó hoạt động (source code, lib, runtime,…)

**Ví dụ thực tế:** Hãy tưởng tượng container như một cái hộp cơm: trong đó có cơm, thịt, rau, muỗng, đũa – tất cả đã chuẩn bị sẵn, chỉ cần mở ra là ăn.

**Trong Docker:** Container được tạo từ image.

## 2. Image (công thức để làm hộp cơm)

**Là cái gì?** Image là bản mẫu (template) để tạo container. Nó chứa tất cả code và môi trường cần thiết để chạy container.

**Ví dụ thực tế:** Hãy tưởng tượng image là công thức nấu cơm. Container là hộp cơm thực tế. Có công thức (image) rồi thì làm ra bao nhiêu hộp cơm (container) cũng được.

**Trong Docker:** Image thường được tải từ Docker Hub (kiểu như siêu thị công thức). Ví dụ, image cho web server nginx.

## 3. Volumes

**Là cái gì?** Volumes là chỗ lưu trữ dữ liệu bên ngoài container. Nó giúp container giữ được dữ liệu kể cả khi tắt hoặc xóa container.

**Ví dụ thực tế:** Hãy tưởng tượng container như cái máy chơi game, còn volume là thẻ nhớ. Game trong máy thì mất khi reset, nhưng dữ liệu (save game) vẫn nằm trên thẻ nhớ.

**Trong Docker:** Volumes thường dùng để lưu database hoặc file mà container cần sử dụng hoặc cập nhật.

# Chạy container đầu tiên

## Bước 1: Mở terminal và chạy lệnh đầu tiên

```bash
docker run -d -p 8080:80 docker/welcome-to-docker
```

> **Lưu ý:** Đây là câu lệnh run đơn giản sử dụng container (hộp cơm) có sẵn. Chưa cần làm gì hết cũng chạy được. Đó là lợi ích của việc sử dụng Docker - không cần cài đặt runtime hay các công cụ khác, cũng chạy được project trong môi trường production. Bài tiếp theo sẽ chi tiết hơn từng thành phần.
