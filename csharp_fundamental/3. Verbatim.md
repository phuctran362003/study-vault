1. Giải thích internal, partial, public

|   |   |   |
|---|---|---|
|internal|partial|public|
|- 1 Class mặc định khi tạp ra là để INTERNAL, che giấu ko cho xuất hiện bên ngoài namespace nó đang  ở<br>- Ko gõ từ INTERNAL đc gọi là DEFAULT, giấu luôn<br>- Vậy nếu mún PUBLIC CLASS vượt ra khỏi căn phòng, vượt ra khỏi namespace, ta để keyword PUBLIC<br>- Che giấu, mở ra, đc gọi chung ENCAPSULATION<br>- Và public/ private/ protected/ default - ko gõ từ khóa/ internal đc gọi chung: ACCESSOR, ACCESS MODIFIER, ACCESS SPECIFIER.|||
