# Basic Cross-Platform Application Programming With .NET

## SDK là gì?

- **SDK** (Software Development Kit) là bộ công cụ dành cho lập trình viên để **biên dịch**, **chạy thử** và **đóng gói** ứng dụng.
- Bên trong thường có **compiler**, **thư viện chuẩn**, **debugger**, **templates** và đôi khi là tài liệu tham khảo.
- Phân biệt với **Runtime**: runtime chỉ đủ để chạy ứng dụng đã build sẵn, còn SDK bao gồm runtime + công cụ phát triển.

## .NET SDK là gì?

- **.NET SDK** = **dotnet CLI** + **compiler** (C#, F#, VB), **Base Class Library**, **project templates**, **MSBuild** và các công cụ hỗ trợ publish.
- Hướng dẫn kiểm tra phiên bản đã cài: `dotnet --info` (trả về danh sách **SDK version**, **Runtime version**, đường dẫn cài đặt).
- .NET SDK hỗ trợ đa nền tảng (Windows, macOS, Linux) nên bạn có thể build và chạy ứng dụng ở bất kỳ OS nào có hỗ trợ.
- Khi cài Visual Studio, thường kèm luôn .NET SDK. Nếu dùng editor khác (VS Code), cần cài thủ công từ `https://dotnet.microsoft.com/download`.

## Quy ước về nơi lưu trữ Source Code – Project

1. **Không đặt tên thư mục/folder/directory** chứa code có dấu tiếng Việt, ký tự **Unicode**, có **khoảng trắng**, **ký tự đặc biệt** – tránh lỗi khi **biên dịch**, **publish** hoặc khi chạy script.
2. **Không lưu code ở ổ đĩa `C:`**. Nên ưu tiên ổ `D:` hoặc `E:` để tránh quyền hạn hệ thống và giảm rủi ro khi cài lại OS.
3. **Cấu trúc thư mục gợi ý** chứa source code:

```text
D:\dev\NET1702\Session01-HelloWorld\src
```

> Gợi ý: đặt tên thư mục theo quy ước `kebab-case` hoặc `PascalCase` để dễ đọc và tương thích khi deploy.

## CLI Commands (.NET Command Line Interface)

- Thuật ngữ cần nhớ: **CLI** (Command Line Interface) và **dotnet CLI**.
- Tạo project console mới:

```powershell
dotnet new console --framework net6.0 --use-program-main
```

- Build project:

```powershell
dotnet build
```

- Chạy trực tiếp project:

```powershell
dotnet run
```

- Khôi phục packages (tự động chạy trong `build`, nhưng nên biết lệnh riêng):

```powershell
dotnet restore
```

- Publish ứng dụng dạng self-contained hoặc framework-dependent:

```powershell
dotnet publish -c Release -r win-x64 --self-contained true
```

> Mẹo: dùng file `global.json` để cố định **SDK version** giữa các thành viên trong dự án, giúp tránh lỗi khác phiên bản.
