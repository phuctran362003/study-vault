# OOP - Object-Oriented Paradigm (Mindset)

## Programming Mindset

- Tư duy lập trình <span style="color:red"><strong>HƯỚNG VỀ ĐỐI TƯỢNG</strong></span>.
- Việc viết code thuần dùng object: luôn tìm và mô tả đúng các object trong bài toán.

## Procedural Programming vs. OOP

- <span style="color:red"><strong>Procedural programming</strong></span> xây dựng chương trình bằng hàm, dữ liệu tách rời, dễ tạo side effect.
- Ví dụ phong cách procedural (C/Pascal):

```c
#include <stdio.h>

int yob = 2003;                 // biến toàn cục

int GetAge(void) {
    return 2024 - yob;
}

void InputYob(void) {
    scanf("%d", &yob);          // mọi hàm đều có thể chỉnh yob
}

int main(void) {
    InputYob();
    printf("Age: %d\n", GetAge());
    return 0;
}

// Side effect: dữ liệu và hàm nằm rời rạc, khó kiểm soát.
```

- OOP gom dữ liệu và hành vi vào cùng object, giảm side effect, giúp bảo trì tốt hơn.

## Đối Tượng (Object) Là Gì?

| Loại giá trị | Mô tả | Ví dụ |
|---|---|---|
| **Single value** | Mang một giá trị duy nhất, không cần mô tả thêm. | `10%`, `3.14`, `50 km/h`, `300000 đ`, `300000 km/s` |
| **Complex value** | Cần nhiều thuộc tính đi kèm để mô tả đầy đủ. | `Messi { Name, Nick, Yob, Home, Goals, ... }`, `Wave Alpha { Model, Volume, Brand, VIN, ... }` |

- Những thực thể cần nhiều thuộc tính/hành vi để mô tả chính là <span style="color:red"><strong>OBJECT</strong></span>.

## Class Và Việc Gom Nhóm Object

- Khi có nhiều object, ta tiến hành <span style="color:red"><strong>CHIA NHÓM</strong></span> theo đặc điểm chung để quản lý.
- Class là tên gọi chung cho một nhóm object giống nhau về thuộc tính và hành vi.
- Ví dụ:
    - Class `Player`: `Name`, `Nick`, `Number`, `Position`.
    - Class `Student`: `Id`, `Name`, `Major`, `Gpa`.
- <span style="color:red"><strong>CLASS</strong></span> đóng vai trò template/blueprint (khuôn) để sinh ra các object cụ thể.

## Quy Trình Tiếp Cận OOP

1. <span style="color:red"><strong>Nhận diện object</strong></span>: tìm các thực thể phức tạp trong bài toán.
2. <span style="color:red"><strong>Gom nhóm thành class</strong></span>: xác định template chứa thuộc tính/hành vi chung.
3. Thiết kế chi tiết class: liệt kê thuộc tính (state) và hành vi (behavior).
4. Tạo object (instantiation) từ class và cho chúng tương tác.

## Bốn Trụ Cột Của OOP

- <span style="color:red"><strong>Encapsulation</strong></span>: gói dữ liệu + hành vi liên quan, che giấu chi tiết thừa.
- <span style="color:red"><strong>Abstraction</strong></span>: chỉ phơi bày phần cần thiết, ẩn phần triển khai.
- <span style="color:red"><strong>Inheritance</strong></span>: tái sử dụng và mở rộng code thông qua quan hệ cha-con.
- <span style="color:red"><strong>Polymorphism</strong></span>: cùng một interface, nhiều cách triển khai.

## Lợi Ích Khi Áp Dụng OOP

- Tổ chức code rõ ràng, dễ chia việc trong team.
- Giảm trùng lặp, tăng tái sử dụng nhờ class và kế thừa.
- Dễ bảo trì vì object tự quản lý dữ liệu của chính nó.
- Linh hoạt mở rộng tính năng bằng cách thêm hoặc kế thừa class mới.

## Ghi Nhớ Quan Trọng

- Luôn gom dữ liệu và logic xử lý vào cùng class để hạn chế side effect.
- Mỗi object nên thể hiện rõ trạng thái (thuộc tính) và hành vi (phương thức).
- OOP là cách mô hình hóa thế giới thực vào phần mềm, giúp bài toán dễ hiểu và dễ mở rộng hơn.
